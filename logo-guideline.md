# 바른자리 로고 가이드라인

## 1) 로고 구성
- 기본형: 심볼 + 워드마크(바른자리)
- 심볼: 라운드 사각 프레임 내부 세로 라인 2개
- 의미: 독립된 공간, 분리된 프라이빗 존, 안정감

## 2) 기본 비율(웹 기준)
- 심볼 크기: 22 x 22px
- 심볼 테두리 두께: 2px
- 심볼 코너 라운드: 6px
- 내부 세로 라인 두께: 2px
- 내부 세로 라인 상하 여백: 3px
- 텍스트 간격(gap): 0.55rem
- 워드마크 폰트 굵기: 800
- 워드마크 자간(letter-spacing): -0.4px

## 3) 색상 시스템
- Primary: #2D6A4F
- Primary Light: #52B788
- 텍스트 기본: #1A1A1A
- 보조 텍스트: #6B7280

### 로고 컬러 사용
- 밝은 배경(헤더):
  - 심볼 테두리: Primary
  - 심볼 내부 라인: rgba(45,106,79,.75)
  - 심볼 내부 배경: linear-gradient(145deg, rgba(82,183,136,.18), rgba(82,183,136,.03))
  - 워드마크: Primary
- 어두운 배경(푸터):
  - 심볼 테두리: rgba(255,255,255,.92)
  - 심볼 내부 라인: rgba(255,255,255,.88)
  - 심볼 내부 배경: linear-gradient(145deg, rgba(255,255,255,.18), rgba(255,255,255,.04))
  - 워드마크: #FFFFFF

## 4) 최소 사이즈
- 웹 최소 권장: 18px(심볼)
- 앱 아이콘/파비콘 최소 권장: 32px
- 심볼+워드마크 조합 최소 폭 권장: 92px

## 5) 여백 규칙(Clear Space)
- 최소 보호 여백: 심볼 폭의 0.5배
- 다른 UI 요소(버튼, 텍스트, 아이콘)와 보호 여백을 침범하지 않도록 유지

## 6) 접근성
- 로고가 링크일 경우 aria-label 제공
- 장식용 심볼 span은 aria-hidden="true" 유지
- 포커스 링 유지(키보드 탐색 가시성)

## 7) 금지 사항
- 로고 비율 임의 변형(가로/세로 늘리기)
- 테두리/라인 삭제 또는 개수 변경
- 과도한 그림자, 외곽선, 글로우 효과 추가
- 배경 대비가 낮아 식별이 어려운 색상 조합 사용
- 워드마크 임의 폰트 변경

## 8) 추천 파일 산출물
- SVG: master(확장/축소 대응)
- PNG: 512, 256, 128, 64, 32
- Favicon: 32, 16

## 9) 웹 구현 기준 클래스
- 헤더 로고: .nav-logo, .logo-mark, .logo-text
- 푸터 로고: .footer-logo .logo-mark, .footer-logo .logo-mark::before, .footer-logo .logo-mark::after

## 10) 운영 체크리스트
- 헤더/푸터 대비 충족 여부 확인
- 모바일 360px 폭에서 로고 가독성 확인
- 다크/라이트 배경에서 식별성 확인
- 링크 포커스 상태 시각적 노출 확인
